PRODUCT NAME
Deal Desk — 3-Engine Real Estate Underwriter

CORE CONCEPT
The app is built around 3 “engines” that pass structured outputs to each other:

1) UNDERWRITING ENGINE
Inputs: all known info about the property + seller (manual user inputs AND enriched/public info AND AVM baselines if available).
Output: “As-is / where-is / today” value RANGE (low/base/high), confidence score, and key value drivers.

2) OFFER CALCULATION ENGINE
Inputs: underwriting output + user settings (profit targets, strategy, fees, risk buffers, market tuning).
Output: recommended offer ladder (3-tier) + assignment fee slider impact + sensitivity summary.

3) OFFER PRESENTATION ENGINE
Inputs: all notes, call summaries/transcripts (manual for MVP) + offer ladder + seller constraints.
Output: a structured, ethical offer presentation plan optimized for win rate (talk track, offer packaging, follow-ups).
Later becomes a premium feature by connecting to GoHighLevel (GHL) via API, but MVP uses manual paste-in notes/transcripts.

ENGINE LOGIC (MUST)

1) UNDERWRITING ENGINE
- Compute AVM base blend:
  - Use whichever AVMs are present (Zillow, Redfin, other).
  - Default weights if present (configurable in code):
    - Zillow 0.45, Redfin 0.35, Other 0.20 (normalize based on availability).
  - If no AVMs present, underwriting must still work using comp placeholder or “manual as-is estimate” input.
- Repairs estimate:
  - rehabPerSqft mapping based on conditionScore (editable constants):
    - 9–10: 8
    - 7–8: 15
    - 5–6: 28
    - 3–4: 45
    - 0–2: 70
  - repairBase = sqft * rehabPerSqft (if sqft missing, flag missing data and set conservative placeholder)
  - Add system-failure adders (simple keyword detection from knownIssues text: HVAC, roof, plumbing, electrical, foundation, waterproofing) — each adds a fixed range bump.
  - Produce repairLow/repairHigh (e.g., base ± 20% plus adders).
- As-is value range:
  - Start from AVM_base (or manual baseline if AVMs missing).
  - Apply condition/repair discount:
    - asIsBase = AVM_base - repairBase - marketabilityDiscount
    - marketabilityDiscount can be a small % of AVM_base based on conditionScore and presence of “failed systems”.
  - asIsLow/asIsHigh as +/- spread based on confidence (low confidence => wider range).
- Confidence score:
  - Score increases with availability of: sqft, beds/baths, yearBuilt, at least 1 AVM, repair clarity, and public info.
  - Output missingData list.
- Output drivers list (5 max) describing what drove value.

2) OFFER CALCULATION ENGINE
- Inputs pull from UnderwritingOutput and OfferSettings.
- Compute investorBuyPrice:
  - base = asIsBase (NOT ARV) for “as-is today” strategy by default for wholesale.
  - incorporate marketCoolingFactorPct reducing base value.
  - investorBuyPrice = (asIsBase * (targetRulePct/100)) - closingCosts - holdingBuffer - riskBuffer
  - if strategy=flip, add “profit target” logic and show a slightly different equation (keep simple but distinct).
- Compute sellerOffer:
  - if strategy=wholesale: sellerOffer = investorBuyPrice - assignmentFee
  - for flip/rental: sellerOffer = investorBuyPrice (assignment fee hidden)
- Offer ladder (3 tiers):
  - “Fast Yes” (highest of the three)
  - “Fair” (middle)
  - “Stretch” (lowest)
  - Ladder should be derived around sellerOffer with +- adjustments and explain “use when…”
- Sensitivity:
  - Display 5 bullet sensitivity items:
    - repairs +10k
    - assignment fee +5k (wholesale only)
    - cooling factor +2%
    - risk buffer +5k
    - holding buffer +2k
- Deal Grade:
  - Simple rule: based on confidence and margin buffers.
